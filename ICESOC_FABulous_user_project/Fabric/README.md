# Fabric/ ‚Äî generated fabric top-level

Everything here is **generated by FABulous** from `fabric.csv` + `Tile/*` definitions. **Do not hand-edit these files** ‚Äî re-run `gen_fabric` instead. The exception is `models_pack.v` which IS hand-curated (and was hand-fixed in this workspace; see below).

## Files

| File | Size | Role |
|---|---|---|
| **`eFPGA_top.v`** | ~43 KB | Top-level fabric wrapper. Exposes external ports (`OPA_I[71:0]`, `OPB_I[71:0]`, `RES0_O/RES1_O/RES2_O[71:0]`, `I_top/O_top/T_top[9:0]`, `Config_accessC[19:0]`, `A_config_C/B_config_C[19:0]`, plus CLK/resetn/SelfWriteData/SelfWriteStrobe/Rx/etc.). Instantiates `eFPGA_Config`, `eFPGA`, BlockRAM, frame pipeline. |
| **`eFPGA.v`** | ~436 KB | The synthesized fabric body ‚Äî every tile instance + every switch matrix + every wire. Mostly a giant netlist. |
| **`eFPGA_Config.v`** | ~3 KB | Configuration FSM (handles BitBang UART input + SelfWrite path). |
| **`BlockRAM_1KB.v`** | ~6 KB | The fabric-side BRAM model. |
| **`config_UART.v`** | ~13 KB | UART receiver for bitstream loading. |
| **`bitbang.v`** | ~3 KB | Serial-to-parallel for the UART. |
| **`Frame_Data_Reg.v`, `Frame_Select.v`** | tiny | Frame-pipeline registers. |
| **`ConfigFSM.v`** | small | Config state machine helper. |
| **`models_pack.v`** | ~10 KB | **Primitive library** ‚Äî defines `config_latch`, `cus_mux21/41/81/161`, `my_buf`, `clk_buf`, `MUX4PTv4`, `MUX16PTv2`, `my_mux2`, `cus_mux*_buf`, `break_comb_loop`. Hand-curated (see below). |

## models_pack.v ‚Äî hand-curated edits

This file **is** hand-edited (one of the few exceptions in this directory):

- **`config_latch (D, E, Q, QN)`** ‚Äî the canonical primitive name used by newer FABulous. Replaced two duplicate `LHQD1` definitions that came from an older FABulous era. The active `*_ConfigMem.v` files instantiate `config_latch`, not `LHQD1`.
- Everything else preserved: `MUX4PTv4`, `MUX16PTv2`, `my_mux2`, `cus_mux41_buf`, `cus_mux81_buf`, `cus_mux161_buf`, `break_comb_loop` ‚Äî these are still used by the project's hand-written BEL Verilog (`LUT4c_frame_config_dffesr.v`, `MUX8LUT_frame_config_mux.v`, `InPass4_frame_config.v`, `OutPass4_frame_config.v`).
- Removed: legacy commented `LHD1_old`, `LHQD1_old` blocks (dead code, left over from primitive renames).

## SoC integration

In the `icesoc_sim/` flow, `eFPGA_top` is instantiated by `icesoc_sim/rtl_icesoc_with_cores/eFPGA_CPU_top.v`. The wiring of the 72-bit `OPA_I`/`OPB_I`/`RES*_O` busses uses a **per-row interleave + bit-reverse** mapping (see `eFPGA_CPU_top.v`'s generate loop) ‚Äî naive concatenation `{W_OPA, E_OPA}` would route operands to the wrong tiles.

In the bare-fabric `Test/` flow, the testbench (`Test/top_tb.v`) does the same interleave directly.
