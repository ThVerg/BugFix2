# Tile/ — per-tile fabric definitions

Each subdirectory here defines one tile type used in the fabric. A tile is a unit of the FPGA grid (one cell in `fabric.csv`'s `FabricBegin` table).

## Tile inventory

| Tile | Role | BELs / function |
|---|---|---|
| **`LUT4AB/`** | Logic tile (the bulk of the fabric) | 8× LUT4 (`LUT4c_frame_config_dffesr` — 4-input LUT + DFF) + 1× MUX8 (`MUX8LUT_frame_config_mux`) |
| **`DSP/{DSP_top,DSP_bot}/`** | DSP super-tile (multiplier) | 1× MULADD BEL (8×8 multiplier + 20-bit accumulator) |
| **`RegFile/`** | Register file tile | 1× `RegFile_32x4` BEL (32 entries × 4 bits, 1 sync write + 2 async/sync read ports) |
| **`RAM_IO/`** | BRAM I/O pad tile | InPass4 / OutPass4 BELs for FAB2RAM_*/RAM2FAB_* signals; bridges fabric to the BRAM |
| **`W_IO/`** | West I/O tile | 10× `IO_1_bidirectional_frame_config_pass` BELs + Config_access |
| **`W_CPU_IO/`, `E_CPU_IO/`** | CPU I/O tiles (top of stem) | 2× InPass4 (OPA, OPB inputs from CPU) + 3× OutPass4 (RES0/1/2 outputs to CPU) |
| **`W_CPU_IO_bot/`, `E_CPU_IO_bot/`** | CPU I/O bottom tiles (transition row Y=9) | Same BELs as the top variants, plus south-side ports for the wider base below |
| **`N_term_*/`** | North termination tiles | No BELs; just chain endpoints (no_config_bits switch matrix) |
| **`S_term_*/`** | South termination tiles | Same |

The N/S termination tiles come in variants (`N_term_single`, `N_term_single2`, `N_term_DSP`, `N_term_RAM_IO`) matching the column types they sit above/below.

## Files inside each tile directory

For a typical `LUT4AB/` (the structure repeats for other tiles):

```
LUT4AB/
├── LUT4AB.v                           ← TILE WRAPPER — instantiates BELs + switch matrix + ConfigMem
├── LUT4AB_switch_matrix.list          ← human-readable connectivity list (source → destination pips)
├── LUT4AB_switch_matrix.csv           ← matrix-form spec (rows=sources, cols=destinations, intersection=mux input index)
├── LUT4AB_switch_matrix.v             ← GENERATED: synthesizable Verilog of the switch matrix
├── LUT4AB_ConfigMem.csv               ← config-bit-to-frame mapping spec
├── LUT4AB_ConfigMem.v                 ← GENERATED: synthesizable ConfigMem (config_latch chain)
└── LUT4c_frame_config_dffesr.v        ← BEL Verilog (the actual LUT+DFF cell)
└── MUX8LUT_frame_config_mux.v         ← BEL Verilog (the 8-to-1 MUX cell)
```

## Generated vs hand-written

- **Hand-written:** the `*.v` BEL files (LUT4c_*, MUX8LUT_*, InPass4_*, OutPass4_*, MULADD.v, RegFile_32x4.v, IO_1_bidirectional_*.v, Config_access.v), the `*_switch_matrix.list` files, and the top-level `[Tile].v` wrapper.
- **Generated by FABulous** (`gen_all_tile`): the `*_switch_matrix.csv`, `*_switch_matrix.v`, `*_ConfigMem.csv`, `*_ConfigMem.v`. **Do not hand-edit these — re-run `gen_all_tile` instead.**

## Tile-level fixes in this snapshot

- **Bot CPU_IO tiles** (`W_CPU_IO_bot/`, `E_CPU_IO_bot/`): the project-side patches add `EE4BEG[0..15] ↔ WW4END[0..15]` (and the symmetric pair) pass-throughs to the `*_switch_matrix.list` files. After the fix to `fabric.csv` (adding the missing `EE4BEG`/`WW4BEG` ports), re-running `gen_all_tile` regenerates the corresponding `.v` and `.csv` to be consistent.
- **Strip stale ConfigBits**: with the FABulous patch in `gen_tile.py`, the regenerated wrappers no longer have empty `.ConfigBits(ConfigBits[20-1:20])` ports, so `strip_configbits.py` is **obsolete**.
- **DSP `_tile.v` siblings** (`DSP/DSP_top/DSP_top_tile.v` and the bot equivalent): legacy duplicate module declarations from older FABulous; **`Test/Makefile`'s `copy_files` target now skips `*_tile.v`** to avoid iverilog duplicate-module errors.
