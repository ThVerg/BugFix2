# ICESOC FABulous user project

A research-grade SoC tape-out target combining a customized RISC-V CPU (FlexBex/Ibex) with an embedded FPGA fabric (eFPGA) generated by [FABulous](https://github.com/FPGA-Research/FABulous). Designed for the [Efabless Caravel/Google MPW](https://efabless.com/open_shuttle_program) open shuttle program (Sky130 130 nm process).

Upstream: [enrica-schmidt/ICESOC_FABulous_user_project](https://github.com/enrica-schmidt/ICESOC_FABulous_user_project). This is a fork with several integration fixes — see the parent [BugFix2 README](../README.md) for the full diff catalogue.

## Architecture (the chip)

Three logical layers:

```
                ┌──────────────────────────────────────────────┐
                │  FlexBex (Ibex variant + custom eFPGA opcode)│   ←  Core 1 = ibex_top
                │  Ibex (vanilla)                               │   ←  Core 2 = flexbex_ibex_core
                ├──────────────────────────────────────────────┤
                │  Wishbone slave + Logic Analyzer probes      │   ← caravel-standard interface
                │  axi_uart, uart_to_mem, sky130_sram (×2)     │
                ├──────────────────────────────────────────────┤
                │  eFPGA (FABulous-generated, inverted-T shape)│   ← user-configurable accelerator
                │  Config FSM + bitbang UART + SelfWrite path  │
                └──────────────────────────────────────────────┘
```

Two CPU cores share an instruction-fetch interconnect (read-only) and a data interconnect (RW), both pointing at two 1KB Sky130 SRAMs (sram_1 + sram_2 = 2KB total). One UART loads bitstream into the eFPGA's Config FSM; another UART loads instructions into SRAM.

The eFPGA is a small, programmable FPGA fabric. Its **inverted-T shape** (narrow stem of 3 columns on top, wider 15-column base on bottom) is what triggered the FABulous geometry bug that this workspace was originally created to investigate.

## eFPGA acceleration interface

FlexBex extends RISC-V with a custom opcode (`0x0B`) for eFPGA acceleration. The CPU drives:

- `eFPGA_operand_a_o[31:0]` — sourced from rs1 register
- `eFPGA_operand_b_o[31:0]` — sourced from rs2
- `eFPGA_operator_o[1:0]` — selects which result to capture (00=a, 01=b, 10=c, 11=write_strobe / config)
- `eFPGA_delay_o[3:0]` — cycles to wait before capture (15 = wait for `eFPGA_fpga_done_i`)
- `eFPGA_en_o`, `eFPGA_write_strobe_o` — control signals

The fabric returns `eFPGA_result_a_i`, `eFPGA_result_b_i`, `eFPGA_result_c_i` (all 32-bit). These map to the user_design's `W_RES0/1/2` (for ibex_top, W side) or `E_RES0/1/2` (for flexbex, E side).

For the shipped `user_design/top.v`:
- W side: `W_RES0 = W_OPA ^ W_OPB`, `W_RES1 = W_OPA & W_OPB`, `W_RES2 = W_OPA | W_OPB`
- E side: `E_RES0 = E_OPA + E_OPB`, `E_RES1 = E_OPA - E_OPB`, `E_RES2 = 32'hDEADBEEF`

## Generating fabric files

The `Tile/*` and `fabric.csv` are the source of truth. Everything in `Fabric/` and most of `Tile/*/[Tile]_switch_matrix.v`, `[Tile]_ConfigMem.v` is **generated**. To regenerate after editing:

```bash
cd ../FABulous
SETUPTOOLS_SCM_PRETEND_VERSION_FOR_FABULOUS_FPGA=0.0.0+local \
  uv run --frozen FABulous \
  -p ../ICESOC_FABulous_user_project \
  run "load_fabric; gen_all_tile; gen_fabric; gen_bitStream_spec"
```

After regeneration: don't forget to re-apply `strip_configbits.py` if you're on an older FABulous version. The current workspace uses a patched FABulous (`FABulous/fabulous/fabric_generator/gen_fabric/gen_tile.py`) that doesn't need stripping.

## Project-side patches (relative to upstream)

These are documented in detail in [../README.md](../README.md) and ../CLAUDE_HANDOFF.md. Summary:

### Geometry-fix related (already in upstream FABulous fork)
- `FABulous/fabulous/geometry_generator/{fabric,tile}_geometry.py` — handle T-shape fabrics, 1×N, N×1 (the original bug).

### Fabric definition
- `fabric.csv` — added missing `EAST,EE4BEG` to W_CPU_IO_bot and `WEST,WW4BEG` to E_CPU_IO_bot.

### SoC integration (real bugs)
- `icesoc_sim/rtl_icesoc_with_cores/eFPGA_CPU_top.v`:
  - Wired `eFPGA_top.resetn` (was unconnected — blocked all bitstream loading).
  - Replaced naive `OPA_I({W_OPA, E_OPA})` with per-row interleave + bit-reverse mapping.
- `icesoc_sim/rtl_icesoc_with_cores/icesoc/icesoc_top.v`:
  - Moved wire decls before first use (iverilog declaration-before-use).
  - Tied off `irq_ack_1_o` / `irq_id_1_o` (ibex_top doesn't drive these).

### Testbench scaffolding (sim-only)
- `eFPGA_CPU_top.v` adds `tb_sw_force/data/strobe` and `tb_drive_ops/W_OPA/W_OPB/E_OPA/E_OPB` overrides for fast deterministic test setup. Default OFF; harmless in synthesis.

### Tile / housekeeping
- `Fabric/models_pack.v` — replaced obsolete `LHQD1` with canonical `config_latch`.
- `Test/Makefile` — `copy_files` skips legacy `*_tile.v`.
- `user_design/top.v` — extended to exercise RegFile tile.
- `Test/top_tb.v` — rewritten for current fabric port signature; added stress + random phases.
